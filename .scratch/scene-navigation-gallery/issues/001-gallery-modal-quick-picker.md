# Issue 001: Gallery Modal Quick Picker

Labels: enhancement, ready-for-agent
Type: AFK

## Parent

`.scratch/scene-navigation-gallery/prd.md`

## What to build

Replace the native center quick-picker with a current-artwork gallery trigger in the top quick-switch controls. Activating the trigger opens a dialog-style gallery that renders all public archive manifest items with the same card facts as the existing archive cards: lazy thumbnail, date, source, title, headline context, and artist influence. The current artwork is visually and accessibly marked. Selecting a different card uses the existing artwork-loading path, closes the gallery, and returns focus to the trigger; selecting the current card closes without a redundant load.

## Acceptance criteria

- [ ] The quick-switch center control is a button/trigger, not a native select, and summarizes the active artwork from existing manifest metadata.
- [ ] The gallery opens from pointer and keyboard activation and exposes dialog semantics with a clear title.
- [ ] The gallery renders all manifest items in manifest order using existing trusted thumbnail and archive-card presentation facts, without fetching every artwork payload.
- [ ] The active artwork is highlighted and exposed with `aria-current` or equivalent current-state semantics.
- [ ] Selecting a different gallery card calls the same load-by-file/index path used by existing archive navigation, closes the gallery, and restores focus to the trigger.
- [ ] Selecting the current gallery card closes the gallery without triggering a redundant artwork load.
- [ ] Escape and the close control close the gallery and restore focus to the trigger.
- [ ] Loading state disables conflicting quick-switch/gallery controls while an artwork load is in flight.
- [ ] Direct artwork links initialize the trigger label and gallery active marker to the linked artwork.
- [ ] Capture mode does not show or open gallery UI.

## Agent Brief

**Category:** enhancement
**Summary:** Replace the native quick-picker with an accessible current-artwork trigger that opens a full public archive gallery modal.

**Current behavior:**
The top quick-switch uses a native select that can only expose text labels. The visual archive grid lives below the fold and is easy to miss, especially because the artwork stage fills the viewport.

**Desired behavior:**
The top quick-switch center control is a discoverable gallery trigger summarizing the active artwork. It opens a dialog-style gallery of all manifest items using archive card presentation facts. The active artwork is marked, selecting another artwork loads it through the existing archive loading path, and closing restores focus to the trigger.

**Key interfaces:**
- Quick-switch presentation facts should describe a trigger label and gallery card facts from public manifest metadata.
- Archive selection by manifest item or file should reuse the existing artwork-load transaction and no-op/close for the current file.
- Runtime shell bindings should support trigger open, close/Escape, and gallery card activation.
- Loading presentation facts should disable conflicting quick-switch/gallery controls while a load is in flight.

**Acceptance criteria:**
- [ ] The quick-switch center control is a button/trigger, not a native select, and summarizes the active artwork from existing manifest metadata.
- [ ] The gallery opens from pointer and keyboard activation and exposes dialog semantics with a clear title.

- [ ] Loading state disables conflicting quick-switch/gallery controls while an artwork load is in flight.
